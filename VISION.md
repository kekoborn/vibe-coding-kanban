# VISION — Claude Kanban

## Что это

Claude Kanban — инструмент для автономной AI-разработки. Не просто трекер задач, а оркестратор рабочего процесса, где Claude Code является исполнителем, а человек — архитектором.

**Принцип:** Человек решает что. Claude решает как.

## Целевой пользователь

Разработчик с несколькими активными проектами, который хочет:
- Оставить Claude работать ночью и утром увидеть готовые фичи
- Не тратить время на ручной запуск каждой задачи
- Контролировать процесс через UI, не углубляясь в терминал

## Ключевые принципы

1. **Маленькие конкретные задачи** — одна задача = один коммит = 2-3 файла максимум
2. **Контекст задачи самодостаточен** — description IS the prompt, без внешних зависимостей
3. **Автономность с контролем** — Claude работает сам, человек проверяет review-колонку
4. **Параллельность** — несколько проектов одновременно, разные PTY

## Рабочий процесс (golden path)

```
/kanban-lead → создать задачи
     ↓
Auto-queue ON + Auto-approve ON
     ↓
Claude работает автономно (или ночью с RalfLoop)
     ↓
Утром: review-колонка = готово к проверке
     ↓
Approve → done, или Return с комментарием
```

## Что НЕ является целью

- Замена полноценного project management (Jira, Linear)
- Enterprise-функции (команды, roles, permissions)
- Поддержка других AI (только Claude Code)
